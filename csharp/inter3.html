C# – Intermediate 3: Access Modifiers

Control the visibility of class members using access modifiers

🔐 What Are Access Modifiers?

Access modifiers define how visible a class or its members are to other parts of the program.

📦 Common Modifiers

  • public – accessible from anywhere
  • private – accessible only within the same class
  • protected – accessible within the same class and derived classes
  • internal – accessible within the same project

🔍 Example


class Person
{
    public string Name;        // Can be accessed from anywhere
    private int age;           // Can be accessed only inside Person

    public void SetAge(int a)
    {
        age = a;
    }

    public void ShowInfo()
    {
        Console.WriteLine($"Name: {Name}, Age: {age}");
    }
}

Note: We use SetAge() to assign a private field safely.

🧠 Logic Tip

  • Always protect internal details using private
  • Use public only when access from outside is needed
  • Encapsulation = hiding internal data and exposing only necessary parts

🧪 Try This

  • Create a class BankAccount with a private balance
  • Add methods to deposit, withdraw, and display balance
  • Use public methods to interact with private data
