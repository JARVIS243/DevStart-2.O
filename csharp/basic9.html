C# – Basic 9: Error Handling

Catch and manage runtime errors using try-catch blocks

🚫 Why Error Handling?

Errors may occur during program execution. Handling them prevents crashes and improves user experience.

🧯 try-catch Block


try
{
    Console.Write("Enter a number: ");
    int num = int.Parse(Console.ReadLine());
    Console.WriteLine("Number is: " + num);
}
catch (FormatException)
{
    Console.WriteLine("Please enter a valid number.");
}

Note: int.Parse() throws an error if input isn't a number.

✅ finally Block (Optional)


try
{
    // Code that might throw an error
}
catch
{
    // Handle the error
}
finally
{
    Console.WriteLine("This always runs.");
}

🧠 Logic Reminder

  • Use try-catch around risky code like user input
  • Catch specific exceptions (e.g., FormatException)
  • finally is useful for cleanup tasks

🧪 Try This

  • Try dividing two numbers and catch divide-by-zero errors
  • Try reading a file that doesn’t exist
  • Use finally to print "End of program"
