C# – Intermediate 7: Properties

Use properties to control access to class fields

🔍 Why Properties?

Properties help manage access to private fields, allowing you to read or update values safely.

⚙️ Example


class Student
{
    private int age;

    public int Age
    {
        get { return age; }
        set
        {
            if (value >= 0)
                age = value;
            else
                Console.WriteLine("Age must be positive.");
        }
    }
}

// Usage
Student s = new Student();
s.Age = 21;  // set
Console.WriteLine(s.Age);  // get

⚡ Auto-Implemented Properties


class Book
{
    public string Title { get; set; }
}

// Usage
Book b = new Book();
b.Title = "DevStart C# Guide";
Console.WriteLine(b.Title);

🧠 Logic Tip

  • Use properties instead of public fields
  • They add a layer of control and validation
  • Auto-properties are short and useful for simple cases
