C# – Intermediate 10: Abstract Classes

Learn how to define base behavior using abstract classes

📌 What is an Abstract Class?

An abstract class cannot be instantiated.

It can have both abstract (unimplemented) and non-abstract (implemented) methods.

📄 Syntax Example


abstract class Animal
{
    public abstract void MakeSound(); // No body

    public void Sleep()
    {
        Console.WriteLine("Sleeping...");
    }
}

class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Dog barks.");
    }
}

// Usage
Dog d = new Dog();
d.MakeSound();  // Output: Dog barks.
d.Sleep();      // Output: Sleeping...

⚠️ Why Use It?

  • To create a base class with some shared behavior
  • To force derived classes to implement specific methods

🧠 Logic Tip

  • Use abstract when some behavior must be implemented by derived classes
  • Abstract classes support fields and constructors (unlike interfaces)
  • You can mix abstract and regular methods inside it

  • Create an abstract class Vehicle with abstract method Drive()
  • Add regular method Refuel()
  • Inherit Car class and implement Drive()
