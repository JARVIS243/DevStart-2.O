Kotlin – Basic 7: Null Safety

Prevent runtime crashes by handling null values safely

❓ What is Null?

null means "no value". Kotlin does not allow variables to be null by default to avoid crashes.


// Safe way to allow null
var name: String? = null

fun main() {
    name = "DevStart"
    println(name?.length) // safe call
}

🔒 Safe Calls

Use ?. to safely access a property or function.


val message: String? = "Hello"
println(message?.length) // works safely

✅ Elvis Operator ?:


val username: String? = null
val finalName = username ?: "Guest"
println(finalName)

‼️ Force Unwrap (Not Recommended)


val name: String? = "DevStart"
println(name!!.length) // Crashes if name is null!

🧠 Logic Tips

  • Use ?. to avoid null pointer exceptions
  • Use ?: to set default values
  • Avoid using !! unless you're sure the value isn't null
