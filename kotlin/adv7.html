Kotlin – Advanced 7: Sealed Classes

Control type hierarchies with limited subclasses

📦 What is a Sealed Class?

A sealed class restricts class hierarchies to a fixed set of subclasses. Great for exhaustive when statements.


sealed class Result

class Success(val data: String) : Result()
class Error(val message: String) : Result()

fun handle(result: Result) {
    when (result) {
        is Success -> println("Success: ${result.data}")
        is Error -> println("Error: ${result.message}")
    }
}

fun main() {
    handle(Success("Data loaded"))
    handle(Error("Something went wrong"))
}

✅ Why Use Sealed Classes?

  • Safe and structured handling of multiple outcomes
  • Compile-time safety with when
  • Used in result handling, state management
