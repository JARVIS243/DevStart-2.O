Kotlin – Advanced 8: Annotations

Metadata for your code

📌 What are Annotations?

Annotations provide metadata to the compiler or runtime. Used for configuration, code generation, or hints.


@Target(AnnotationTarget.FUNCTION)
annotation class LogExecution

@LogExecution
fun sayHello() {
    println("Hello from DevStart!")
}

Note: You'll need reflection or frameworks to act on annotations.

✅ Common Annotations

  • @Deprecated – Marks code as outdated
  • @JvmStatic, @JvmOverloads – Interop with Java
  • @Serializable – Marks class for serialization
