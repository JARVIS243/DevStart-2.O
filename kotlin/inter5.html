Kotlin – Intermediate 5: Visibility Modifiers

Control where variables and functions can be accessed from

🔐 Visibility Levels

  • public – visible everywhere (default)
  • private – visible only inside the class or file
  • protected – like private, but visible in subclasses
  • internal – visible inside the same module

🔍 Example


class BankAccount {
    private var balance = 0

    fun deposit(amount: Int) {
        balance += amount
    }

    fun showBalance() {
        println("Balance: $balance")
    }
}

fun main() {
    val acc = BankAccount()
    acc.deposit(100)
    acc.showBalance()
    // acc.balance = 500 // ❌ Error: balance is private
}

🧠 Logic Tips

  • Use private to protect sensitive data
  • Keep things public only if needed outside
  • protected helps subclasses access without exposing outside

🧪 Try This

  • Make a User class with username public and password private
  • Try accessing both from main and see what works
