Optimize performance by transferring resources instead of copying

‚ö° What is Move Semantics?

Move semantics allow the resources of temporary objects to be transferred rather than copied, improving efficiency.

üßë‚Äçüíª Example: Move Constructor


#include <iostream>
#include <vector>
using namespace std;

class Buffer {
public:
    vector<int> data;

    // Constructor
    Buffer(int size) : data(size) {}

    // Move constructor
    Buffer(Buffer&& other) noexcept : data(move(other.data)) {
        cout << "Move constructor called" << endl;
    }

    // Copy constructor
    Buffer(const Buffer& other) : data(other.data) {
        cout << "Copy constructor called" << endl;
    }
};

int main() {
    Buffer b1(1000);
    Buffer b2 = move(b1);  // Calls move constructor

    return 0;
}

üîç Notes

  • Move semantics improve performance for temporary objects
  • Use std::move() to enable move operations
  • Define move constructors and move assignment operators
