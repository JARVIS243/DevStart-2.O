Python: Advanced Lesson 2 – Exception Handling

Handle errors gracefully and keep your program running smoothly

⚠️ What is an Exception?

Exceptions are errors that occur during program execution. If not handled, they can crash the program.

🛑 Common Example


x = int(input("Enter a number: "))
y = 10 / x
print("Result:", y)

If x = 0, you’ll get a ZeroDivisionError

✅ Handling Errors with try-except


try:
    x = int(input("Enter a number: "))
    y = 10 / x
    print("Result:", y)
except ZeroDivisionError:
    print("You can't divide by zero!")

📌 Multiple Exceptions


try:
    num = int(input("Enter a number: "))
    print(10 / num)
except ValueError:
    print("That's not a number!")
except ZeroDivisionError:
    print("Cannot divide by zero!")

📦 Using finally


try:
    f = open("data.txt")
    print(f.read())
except FileNotFoundError:
    print("File not found.")
finally:
    print("Done trying to open file.")

💡 Raise Custom Exceptions


age = int(input("Enter your age: "))
if age < 0:
    raise ValueError("Age can't be negative")

🧪 Try This

  • Write a calculator that handles wrong input and division by zero
  • Try opening a file and handle if it doesn't exist
  • Raise an error if a number is out of a certain range

📌 Summary

  • try-except prevents crashes
  • Use specific exceptions like ValueError, ZeroDivisionError
  • finally runs no matter what
  • raise throws your own error
