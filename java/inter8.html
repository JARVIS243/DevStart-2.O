Java: Intermediate Lesson 8 – Abstract Classes

Define a blueprint class that cannot be directly instantiated

🧩 What is an Abstract Class?

An abstract class in Java is a class that cannot be instantiated. It can have both abstract (no body) and regular methods. It's used to define a common structure for subclasses.

📘 Syntax


abstract class Animal {
    abstract void makeSound();  // abstract method

    void sleep() {
        System.out.println("Sleeping...");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Dog barks");
    }
}

🚀 Using It


public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.makeSound();  // Dog barks
        d.sleep();      // Sleeping...
    }
}

📌 Key Points

  • Use abstract keyword to define abstract class or method
  • You cannot create an object of an abstract class
  • Child classes must implement all abstract methods

🧪 Try This

  • Create an abstract class Shape with an abstract method area()
  • Implement it in Circle and Square classes
  • Call their area() methods

📌 Summary

  • abstract class = partial blueprint
  • Can include normal and abstract methods
  • Cannot be directly instantiated
  • Forces subclasses to implement key functionality
