Java: Advanced Lesson 10 – Generics

Write flexible and type-safe code using generics

🔁 What are Generics?

Generics allow you to write code that works with different types, while ensuring type safety at compile time. They're commonly used with collections like ArrayList, HashMap, etc.

📘 Example: Without vs With Generics


// Without generics
ArrayList list = new ArrayList();
list.add("Hello");
String s = (String) list.get(0);  // requires casting

// With generics
ArrayList<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0);  // no casting needed

📦 Generic Class


class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

public class Main {
    public static void main(String[] args) {
        Box<String> stringBox = new Box<>();
        stringBox.set("DevStart");
        System.out.println(stringBox.get());

        Box<Integer> intBox = new Box<>();
        intBox.set(123);
        System.out.println(intBox.get());
    }
}

📌 Benefits of Generics

  • Type safety – catch errors at compile time
  • No need for explicit type casting
  • Code reusability – same code works with different types

🧪 Try This

  • Create a generic class called Pair<K, V> that stores a key and a value
  • Use it with different data types (e.g., String, Integer)
  • Print out the stored key-value pairs

📌 Summary

  • Generics help you write cleaner, reusable, and safer code
  • Use <T>, <E>, <K, V> for generic classes and methods
  • Commonly used in collections
