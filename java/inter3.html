Java: Intermediate Lesson 3 – Method Overloading

Use the same method name with different parameters

📘 What is Method Overloading?

Method Overloading means defining multiple methods with the same name but different parameters. Java decides which version to run based on arguments.

🧠 Example


public class Calculator {
    // add(int, int)
    int add(int a, int b) {
        return a + b;
    }

    // add(double, double)
    double add(double a, double b) {
        return a + b;
    }

    // add(int, int, int)
    int add(int a, int b, int c) {
        return a + b + c;
    }

    public static void main(String[] args) {
        Calculator c = new Calculator();
        System.out.println(c.add(2, 3));         // 5
        System.out.println(c.add(2.5, 4.5));     // 7.0
        System.out.println(c.add(1, 2, 3));      // 6
    }
}

🔍 Rules for Overloading

  • Method name must be the same
  • Parameter count or type must differ
  • Return type alone is not enough to overload

🧪 Try This

  • Create a Printer class
  • Add methods print(String) and print(int)
  • Add one more: print(String, int)

📌 Summary

  • Method Overloading = same name, different parameters
  • Improves readability and reusability
  • Java chooses the correct version during compile-time
