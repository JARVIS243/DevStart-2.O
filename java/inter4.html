Java: Intermediate Lesson 4 – Method Overriding

Redefine a method of the parent class in the child class

🔁 What is Method Overriding?

Method Overriding means providing a new version of a method from the parent class inside the child class. It lets you change or extend the behavior of inherited methods.

📘 Example


class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.sound();  // Output: Dog barks
    }
}

📌 Key Points

  • Method name, return type, and parameters must match exactly
  • Use @Override annotation to indicate overriding
  • Overriding happens in runtime (dynamic binding)

🧪 Try This

  • Create a Vehicle class with method start()
  • Override start() in a Bike class
  • Create objects of both and call start()

🔄 Overloading vs Overriding

  • Overloading: Same class, same method name, different parameters
  • Overriding: Parent-child classes, same method signature

📌 Summary

  • Overriding lets a subclass modify parent class behavior
  • Helps achieve runtime polymorphism
  • Must match method name, return type, and parameters exactly
