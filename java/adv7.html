Java: Advanced Lesson 7 – Thread Lifecycle

Understand the stages a thread goes through from start to end

🔄 What is the Thread Lifecycle?

A thread in Java goes through multiple states from its creation to termination. Understanding these states helps manage thread behavior effectively.

🌀 Thread States

  1. New: Thread object is created but not started
  2. Runnable: Thread is ready to run (after start())
  3. Running: Thread is executing
  4. Blocked/Waiting: Thread is waiting for a resource or another thread
  5. Terminated: Thread finishes or is stopped

📘 Example


class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t = new MyThread();
        System.out.println("State: " + t.getState());  // NEW
        t.start();
        System.out.println("State: " + t.getState());  // RUNNABLE
    }
}

🧠 Thread Methods Related to Lifecycle

  • start() – moves from NEW → RUNNABLE
  • run() – enters RUNNING
  • sleep() – enters TIMED_WAITING
  • join() – causes current thread to wait
  • getState() – returns current state

📌 Summary

  • Threads have a clear lifecycle: New → Runnable → Running → Waiting → Terminated
  • Use getState() to monitor threads
  • Thread management helps build stable concurrent apps

🧪 Try This

  • Create a thread and use getState() to print its state before and after start()
  • Use sleep() and observe state
