Java: Intermediate Lesson 5 – Access Modifiers

Control visibility of classes, variables, and methods

🔐 What Are Access Modifiers?

Access modifiers in Java control where a class or member (variable/method) can be accessed from.

📦 Types of Access Modifiers

Modifier Description Access Scope
public Accessible everywhere Any class
private Accessible only within the same class Only inside declared class
protected Accessible in the same package and subclasses Same package + subclass
(default) No modifier → package-private Same package only

🧠 Example


public class Person {
    public String name = "Aju";
    private int age = 20;
    protected String city = "Kochi";

    public void showName() {
        System.out.println("Name: " + name);
    }

    private void showAge() {
        System.out.println("Age: " + age);
    }

    protected void showCity() {
        System.out.println("City: " + city);
    }
}

💡 Key Notes

  • Use private for internal details like passwords, age, etc.
  • public for anything meant to be accessed from anywhere
  • protected helps when using inheritance
  • No modifier = accessible only in the same package

🧪 Try This

  • Create a BankAccount class
  • Make balance private, and provide public methods deposit() and getBalance()
  • Try accessing balance directly (should fail)

📌 Summary

  • Access modifiers help manage security and structure
  • private = most restricted, public = fully open
  • Use wisely to protect data and code
