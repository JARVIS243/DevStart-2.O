Java: Intermediate Lesson 10 – Packages

Organize your Java classes into structured folders

📦 What is a Package?

A package is a namespace that organizes related classes and interfaces. Think of it like a folder structure in your project.

📘 Creating a Package


// File: MyClass.java
package mypackage;

public class MyClass {
    public void sayHello() {
        System.out.println("Hello from MyClass in mypackage!");
    }
}

🚀 Using a Package


// File: Main.java
import mypackage.MyClass;

public class Main {
    public static void main(String[] args) {
        MyClass obj = new MyClass();
        obj.sayHello();
    }
}

📌 Built-in Packages

  • java.util – for collections like ArrayList, HashMap
  • java.io – for file handling
  • java.net – for networking

🧪 Try This

  • Create a package mathutils
  • Add a class Calculator with methods like add() and subtract()
  • Use the Calculator class in another file by importing the package

💡 Best Practices

  • Use lowercase names for packages (e.g., myapp.tools)
  • Group related classes in meaningful packages
  • Keep file structure consistent with package names

📌 Summary

  • package keyword defines a class's package
  • import keyword is used to use classes from other packages
  • Packages help organize and avoid name conflicts
